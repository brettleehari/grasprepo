What happens under the hood of coding agents?

Every modern coding assistant — Cursor, Aider, Claude Code, Copilot — needs to understand your entire codebase before it can help you. But how? They can't just read every file. Here's the pipeline they run behind the scenes, and exactly what grasprepo shows you:

1

Parse the AST

Every source file is parsed into an Abstract Syntax Tree using Tree-sitter — the same parser that powers VS Code, Neovim, and GitHub's code search. This extracts every class, function, method, and variable definition — the structural skeleton of your code.

2

Build the Dependency Graph

For every symbol defined in one file and referenced in another, an edge is drawn. This produces a directed graph where files point to the files they depend on. The result is a map of how information flows through your codebase.

3

Rank by Importance (PageRank)

The same algorithm Google uses to rank web pages is applied to your dependency graph. Files that are heavily referenced by other important files rise to the top. This tells the agent which files matter most — and which to include in its context window.

4

Measure Code Churn

Git history reveals which files change most often, who changes them, and how much. High churn signals active development, instability, or tech debt. Agents use this to understand what's "hot" in the codebase right now.

5

Surface Hotspots

The most dangerous code lives at the intersection: files that are both highly connected (many things depend on them) and frequently changing. These are your highest-risk files — where bugs have the biggest blast radius.
